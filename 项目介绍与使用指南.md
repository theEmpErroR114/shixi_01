# 课程习题测验系统 — 项目介绍与使用指南

## 一、项目简介

### 1.1 项目背景

课程习题测验系统是一个面向课程教学的在线练习与测验平台。在传统教学模式下，教师出题、组卷、批改与统计工作量大、周期长，学生缺乏随时随地自主练习和即时获得反馈的途径。本系统通过信息化手段，将课程练习与测验环节线上化、数据化，提升教学效率与反馈质量。

### 1.2 系统特点

- **三端分离**：管理员、教师、学生三类角色拥有独立的登录入口和功能空间，权限互不交叉
- **题库管理**：支持单选题、多选题、判断题、填空题、简答题五种题型
- **灵活组卷**：教师可从题库中选题组卷，设置分值、时长，发布正式测验
- **自主练习**：学生可按课程随机抽题练习，即时获得对错反馈
- **成绩统计**：教师可查看学生的练习和测验情况，辅助教学决策

### 1.3 技术栈

| 类别 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Spring Boot 4.0.7 (Java 26) | 核心后端框架，提供 RESTful API |
| 持久层 | MyBatis 4.0.1 | SQL 映射框架，操作数据库 |
| 数据库 | MySQL 8.x | 关系型数据库，存储所有业务数据 |
| 认证方式 | Session + Cookie | 基于会话的登录认证，无需额外安全框架 |
| 前端 | HTML + Tailwind CSS + 原生 JavaScript | 静态页面 + Ajax 调用后端接口 |
| 构建工具 | Maven | 项目构建与依赖管理 |

## 二、系统启动

### 2.1 环境要求

- JDK 26+
- Maven 3.8+
- MySQL 8.x（需提前创建数据库）

### 2.2 启动步骤

```bash
# 1. 构建项目
mvn -f exam-system-backend/pom.xml clean package -DskipTests

# 2. 启动服务
java -jar exam-system-backend/target/exam-system-backend-1.0.0.jar

# 3. 访问系统
# 浏览器打开 http://localhost:8080/login.html
```

首次启动时，系统会自动初始化种子数据（管理员、教师、学生账号各若干，课程2门，习题3道，试卷1份）。

## 三、管理员使用指南

管理员是系统权限的最高层级，负责**账号体系的维护**和**课程管理**，不参与出题、组卷或做题。

### 3.1 登录

1. 打开 `http://localhost:8080/login.html`
2. 选择角色为「管理员」
3. 输入账号 `admin`，密码 `123456`
4. 点击登录，进入管理员仪表盘

### 3.2 仪表盘

仪表盘展示系统核心统计数据：

- **老师总数**：系统中注册的教师账号数量
- **学生总数**：系统中注册的学生账号数量

### 3.3 老师账号管理

进入「老师账号管理」页面，可进行以下操作：

| 操作 | 说明 |
|------|------|
| **搜索** | 输入工号、姓名或手机号，实时筛选教师列表 |
| **筛选** | 按状态（启用/禁用）筛选 |
| **新增** | 点击「新增老师」，填写工号、姓名、性别、学科、手机号，初始密码默认为 `123456` |
| **编辑** | 修改教师的姓名、性别、学科、手机号（工号不可修改） |
| **重置密码** | 将教师密码重置为默认密码 `123456` |
| **启用/禁用** | 切换教师账号状态，禁用后该教师无法登录系统 |

### 3.4 学生账号管理

进入「学生账号管理」页面，功能与老师账号管理类似：

| 操作 | 说明 |
|------|------|
| **搜索** | 按学号、姓名、手机号搜索 |
| **筛选** | 按班级、状态筛选 |
| **新增** | 填写学号、姓名、性别、班级、手机号，初始密码默认 `123456` |
| **编辑** | 修改学生信息 |
| **重置密码** | 重置为默认密码 |
| **启用/禁用** | 切换学生账号状态 |

### 3.5 课程管理

进入「课程管理」页面，管理系统中的课程：

| 操作 | 说明 |
|------|------|
| **搜索** | 按课程名称搜索 |
| **新增** | 填写课程名称和描述 |
| **编辑** | 修改课程名称和描述 |
| **删除** | 删除课程（若该课程下有题目或试卷，将拒绝删除并提示数量） |

> **注意**：课程是题库和试卷的分类维度。教师出题时需选择所属课程，学生练习时可按课程筛选。因此新增课程应由管理员在教学开始前完成。

### 3.6 退出登录

点击右上角头像下拉菜单，选择「退出登录」返回登录页。

## 四、教师使用指南

教师是教学内容的管理者，负责**习题管理、试卷管理**，并**查看学生的练习和测验情况**。

### 4.1 登录

1. 打开登录页，选择角色为「老师」
2. 输入账号（如 `teacher_wang`）和密码 `123456`
3. 登录后进入教师仪表盘

### 4.2 仪表盘

仪表盘展示四项关键指标：

- **习题总数**：该教师创建的题目数量
- **试卷总数**：该教师创建的试卷数量（含未发布草稿数）
- **已发布测验**：当前状态为「已发布」的试卷数量
- **学生总数**：系统中的学生总数

### 4.3 习题管理

进入「习题管理」页面，管理个人题库：

#### 查看与筛选

- 左侧显示课程树，点击课程可筛选该课程下的题目
- 支持按题型（单选/多选/判断/填空/简答）和难度（易/中/难）筛选
- 支持按关键词搜索题干内容

#### 新增习题

1. 点击「新增习题」按钮
2. 选择所属课程
3. 选择题型：
   - **单选题（type=1）**：添加 A/B/C/D 选项，勾选一个正确答案，答案自动从选项生成
   - **多选题（type=2）**：添加多个选项，勾选多个正确答案，答案自动从选项生成
   - **判断题（type=3）**：填写参考答案（"对"或"错"）
   - **填空题（type=4）**：填写参考答案
   - **简答题（type=5）**：填写参考答案和解析
4. 选择难度等级
5. 填写答案解析（可选）
6. 点击保存

#### 编辑/删除习题

- 点击题目列表中的「编辑」可修改题目内容
- 点击「删除」可删除题目（已关联到试卷的题目删除时会同步清理关联关系）

### 4.4 试卷管理

进入「试卷管理」页面，管理试卷的完整生命周期：

#### 创建试卷

1. 点击「新建试卷」
2. 填写试卷名称、选择所属课程
3. 设置总分和考试时长（分钟）
4. 保存后进入组卷阶段

#### 组卷（编辑试卷题目）

1. 在试卷卡片上点击「组卷」
2. 从题库中选择题目添加到试卷
3. 为每道题设置分值
4. 所有题目分值之和必须等于试卷总分，否则无法发布

#### 试卷状态流转

```
未发布（草稿） → 已发布 → 已回收
```

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| **未发布（status=0）** | 草稿状态，可编辑 | 编辑信息、组卷、发布、删除 |
| **已发布（status=1）** | 学生可参加测验 | 回收 |
| **已回收（status=2）** | 测验结束，学生不可再参加 | 查看成绩统计 |

> **注意**：只有「未发布」状态的试卷可以编辑内容和组卷。发布后不可修改。

### 4.5 查看学生情况

进入「学生管理」页面，查看学生的练习和测验数据：

- 查看每位学生的练习次数、正确率
- 查看学生的测验成绩和排名
- 按课程、班级筛选学生数据

### 4.6 退出登录

点击右上角头像下拉菜单，选择「退出登录」。

## 五、学生使用指南

学生是系统的终端使用者，通过系统进行**自主练习**和**参加测验**，并**查看成绩**。

### 5.1 登录

1. 打开登录页，选择角色为「学生」
2. 输入账号（如 `stu_zhang`）和密码 `123456`
3. 登录后进入学生仪表盘

### 5.2 仪表盘

仪表盘展示：

- **练习总次数**：该学生的累计练习次数
- **练习正确率**：所有练习的总体正确率
- **最近测验成绩**：最近一次测验的得分
- **待参加测验**：已发布但尚未参加的试卷列表

### 5.3 抽题练习

进入「抽题练习」页面，进行自主练习：

1. **配置练习**：
   - 选择课程（如「Java程序设计」）
   - 选择题型（可选，不选则随机）
   - 选择难度（可选，不选则随机）
   - 设置题目数量
2. **开始练习**：系统从题库中随机抽取符合条件的题目
3. **作答**：逐题作答
   - 单选题：点击选择一个选项
   - 多选题：勾选多个选项
   - 判断题：选择「对」或「错」
   - 填空题：输入答案文本
   - 简答题：输入答案文本
4. **提交并查看结果**：提交后即时显示对错判断和答案解析

> **说明**：练习模式下，题目不显示答案和解析，提交后才展示。练习记录会保存到系统中，供教师查看。

### 5.4 参加测验

进入「参加测验」页面，参加教师发布的正式测验：

1. 在「待参加测验」列表中选择一份试卷
2. 点击「开始测验」，系统记录开始时间
3. 在规定时间内完成所有题目
4. 点击「提交试卷」（或倒计时结束后自动提交）
5. 系统自动评分并记录成绩

> **注意**：
> - 测验有时间限制，请在倒计时结束前完成
> - 提交后不可修改答案
> - 已回收的试卷无法再参加

### 5.5 查看成绩

进入「我的成绩」页面，查看历史练习和测验记录：

- **练习记录**：每次练习的题目、作答、对错详情
- **测验记录**：每次测验的总分、各题得分、正确答案对照
- 支持按课程筛选，查看不同课程的成绩

### 5.6 退出登录

点击右上角头像下拉菜单，选择「退出登录」。

## 六、系统使用流程总览

```
管理员创建账号和课程
        ↓
教师登录 → 出题（按课程分类）→ 组卷 → 发布测验
        ↓
学生登录 → 日常抽题练习 / 参加正式测验 → 查看成绩
        ↓
教师查看学生练习和测验数据 → 调整教学策略
```

### 完整使用流程示例

1. **管理员**登录系统，创建课程「Java程序设计」，创建教师账号 `teacher_wang` 和学生账号 `stu_zhang`
2. **教师** `teacher_wang` 登录，在「Java程序设计」课程下创建多道习题（单选、多选、判断等）
3. **教师**创建试卷「Java第一章测验」，从题库中选题组卷，设置分值，发布测验
4. **学生** `stu_zhang` 登录，在练习模块选择「Java程序设计」课程进行随机抽题练习
5. **学生**在测验模块看到教师发布的「Java第一章测验」，点击参加并完成答题
6. **学生**在成绩模块查看测验得分和答题详情
7. **教师**在学生管理模块查看全班的练习正确率和测验成绩
